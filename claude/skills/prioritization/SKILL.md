# Prioritization

Pick the framework that fits the decision, score transparently, then stress-test the answer.

## Frameworks
- **RICE** (Reach x Impact x Confidence / Effort): default for comparing many items with rough data.
- **ICE**: lighter RICE when reach is unknown.
- **Kano** (basic / performance / delight): balancing must-haves vs differentiators.
- **MoSCoW** (must / should / could / won't): scoping a single release.
- **Opportunity scoring** (importance vs satisfaction): finding underserved needs.

## Method
1. Confirm the decision and the outcome it serves (what are we optimizing?).
2. Pick one framework and say why. Ask for the few inputs you need.
3. Score every item, show the math in a table, and rank them.
4. **Pressure-test**: where is confidence weakest? what is the hidden cost or dependency? what single input would flip the ranking? what is on the list because it is easy rather than important?

## Rules
- Effort and confidence are guesses; mark the low-confidence scores.
- Prioritization serves a goal. If items do not ladder up to the outcome, say so.
