# Portable Claude Code statusline

A self contained, cross platform (macOS and headless Linux) Claude Code
statusline. It renders across two lines:

* **Line 1 (identity)**: project name (always shown, blue, derived from the git
  repo so it stays stable no matter where you cd), an optional purple `/subdir`
  suffix when you are below the repo root, a magenta `⑂ <name>` section that
  appears only inside a linked git worktree, the git branch with a clean/dirty
  dot (green when clean, red `●N` with the changed-file count when dirty) and an
  optional `↑ahead ↓behind` versus upstream, then lines changed
  (`+added/-removed`), then the model with its reasoning effort and an
  extended-thinking dot when enabled, then session wall-clock time.
* **Line 2 (usage)**: context percentage, your Claude subscription usage (5 hour
  and weekly) with progress bars, pace markers, and reset times, then session
  cost.

Segments are separated by a gray bullet. On narrow terminals the line
collapses: below 90 columns the secondary metrics (cost, lines, duration,
ahead/behind) drop; below 65 the bars, pace markers, reset times, and the whole
weekly segment drop too. Every segment has a `SHOW_*` toggle in
`statusline-config.txt` (`RESPONSIVE=0` disables the collapse).

Claude Code renders each printed line as its own status row (see the official
docs, "Display multiple lines").

## Where the data comes from

Everything is read from the JSON payload Claude Code pipes to the script on
stdin. That includes the subscription usage numbers: Claude Code exposes them as
a `rate_limits` object (`five_hour` and `seven_day`, each with a
`used_percentage` and a `resets_at` epoch) for Claude.ai Pro and Max sessions,
starting from the first API response in a session. See the "Available data"
section of the Claude Code statusline docs.

Because the numbers track whatever account Claude Code itself is logged into,
switching accounts needs nothing here: the usage segment follows Claude Code
automatically. There is no cache, no network call, no cookie, and no external
dependency. The renderer is pure bash plus the standard `date` and `grep`.

Each `rate_limits` window can be independently absent (empty early in a session,
or on non-subscription auth). When a window is missing the usage segment shows
`~` and the weekly segment is simply omitted, so the line degrades cleanly.

## Pieces

| File | Role |
|------|------|
| `statusline.sh` | The renderer. Claude Code pipes a JSON payload to it on stdin; it prints the two status lines. Pure bash. |
| `statusline-config.txt` | Feature toggles and colors (which segments show, 12 vs 24 hour clock, and so on). |

## Install on a new machine

`install.sh` in the Dotfiles root symlinks this directory to
`~/.claude/statusline` and points `settings.json`'s `statusLine` at
`~/.claude/statusline/statusline.sh`. Nothing else to wire, on macOS or on a
headless Linux box.

The usage segment needs Claude Code recent enough to emit `rate_limits` on
stdin (any current release). If you run an older build, or use API/console auth
rather than a Claude.ai subscription, the usage and weekly segments show `~` and
everything else still works.

## Config

Edit `statusline-config.txt`. Toggles are self describing (`SHOW_USAGE`,
`SHOW_WEEKLY`, `USE_24_HOUR_TIME`, `COLOR_MODE`, per element colors, and so on).
Set `SHOW_USAGE=0` to drop the usage and weekly segments and get a line with
just directory, branch, model, and context.

## Troubleshooting

* **Usage shows `~`**: Claude Code did not provide `rate_limits` on stdin. Common
  reasons: the session has not made its first API response yet, the account is
  not a Claude.ai Pro/Max subscription, or the Claude Code build predates the
  `rate_limits` field. Nothing to fix in this repo.
* **Reset time looks wrong**: it renders in the box's local timezone. On a UTC
  VPS you will see UTC. Set the machine timezone if you want local.

## History

This started as the script generated by the "Claude Usage" macOS menu bar app
(HamedElfayome), which scraped usage from claude.ai behind a session cookie. It
was first detached from that app to run anywhere, then rewritten to read the
`rate_limits` data Claude Code now provides directly, which removed the cookie,
the `curl-impersonate` dependency, and the background refresh entirely.
